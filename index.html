Custom 3D-Print Order Pipeline

eBay store · Personalized printed goods

What it is: A two-platform decoupled pipeline that turns eBay order emails into print-ready STL files for a Bambu printer.

Apps Script polls Gmail, regex-parses eBay order notifications, and writes structured rows to a Google Sheet with status To Be Printed. A scheduled GitHub Actions workflow reads the sheet, generates parameterized STLs via OpenSCAD, and commits them back to the repo for human review before printing.

Architecture diagram
Architectural decision

Fully deterministic at runtime — no LLM in the data path. Gemini was used at development time to write the regex parsers and OpenSCAD templates, but the running system is plain code. In a pipeline where wrong output costs filament, determinism is more valuable than flexibility.

Two-stage human gate: Stage 1 validates that the customization matches buyer intent. Stage 2 dispatches to the printer manually, deliberately unautomated — automating it without a print queue manager would cause the resource collisions it's meant to prevent on a single-printer setup.

AI-Assisted YouTube Reel Pipeline

300+ Dohas channel · Publishing every 3 days

What it is: A hybrid two-stage pipeline that turns raw video reels into published YouTube content with human-reviewed AI-drafted captions.

Stage 1 runs on Make.com (low-code orchestration): Gemini transcribes each reel, drafts a caption and hook, writes the row to a Google Sheet, and moves the reel to a processed folder for idempotency. Stage 2 runs as a Python job on GitHub Actions: reads approved rows from the sheet, fetches the matched reel from Drive, and posts to YouTube every 3 days.

Architecture diagram
Architectural decision

Designed as augmentation, not full automation — early tests showed caption quality drift when humans were removed. Low-code where the work is exploratory and creative; code where the work is operational and repeatable.

State-via-location idempotency: a reel's folder location is its processing state. No separate database, no flag column to maintain, no race conditions between scheduler and worker.

Workforce Digest

Concept prototype — SMB HR analytics

What it is: A weekly Monday brief that turns HRIS data into 3–4 things HR managers should act on, with full transparency about how each alert was derived.

Built end-to-end: rule-based detection engine, transparent provenance for every alert, multi-layer validation framework (cross-signal corroboration, outcome-based checks, randomized holdout), and a 10-case Python eval suite for catching regressions.

Architectural decision

Designed the validation layer specifically to address the prevention paradox — when an intervention succeeds, you can never know if the alert was right or noisy. Cross-signal corroboration plus outcome-based checks plus a randomized holdout were necessary because no single check could distinguish a genuinely caught risk from a false positive that resolved on its own.

Cross-Vendor HCM Pain-Point Analysis

Oracle FDI · Workday · SAP SuccessFactors · UKG · Dayforce · ADP · Rippling · BambooHR · Gusto

What it is: A synthesis of customer-reported pain points across nine HCM vendors, drawing from 17+ research platforms. This 2026 update specifically examines the friction between local statutory requirements and global platform capabilities.

The first three sections provide a structured AI-augmented research pass on top vendor-specific pain points. Sections 4–6 pivot to a strategic analysis of where "Global" HCM platforms are currently failing the "Local" reality of HR teams.

How this was made

An AI-directed synthesis using Claude and Gemini for source aggregation and citation-checking. The domain expertise is in the framing: knowing that vendor-neutral compliance and the "Global vs. Local" reporting gap are the real indicators of platform longevity, rather than just feature counts.

Three positions the analysis takes:

  • The "Global-Local" Compliance Gap: While all nine vendors claim global compliance, customer friction is highest in local statutory reporting (specifically in the EU and APAC). The "Global" layer often lacks the granularity needed for local labor law adherence, forcing users back into manual spreadsheets.
  • Payroll is no longer a "Silo" pain: Payroll errors are increasingly cited as symptoms of poor Time & Labor or Benefits integration rather than standalone engine failures. The analysis shows that integration latency between core HCM and payroll is the single biggest driver of customer churn in 2026.
  • Reporting as a Specialized Skill: The data reveals a universal complaint that HCM analytics tools (FDI, Workday Prism, etc.) still require "Power Users." The gap between a manager's question and the system's "Out-of-the-box" report is the #1 reason for secondary-tool adoption (Visier, Tableau).

Each position is defended in the full analysis with sourced evidence per vendor and per pain category.
